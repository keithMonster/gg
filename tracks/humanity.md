## 驱动问题 (Driving Questions)

- 在架构决策中最常被低估的认知偏差是哪些？

  - 可得性启发 (availability heuristic)
  - 幸存者偏差
  - 现状偏误
- gg 的 7 步硬流程能对抗哪几种？对抗不了的那几种，怎么显式标注？

### DQ-2. 动机与激励的错位
- "团队说想要 A，实际上真的想要 A 吗？"——意图与表达的鸿沟在架构决策中如何处理？
- 当 Keith 给 gg 一个决策请求时，他真正想要的是 "一个答案" 还是 "被挑战"？
- gg 怎么分辨这两者？

### DQ-3. 沟通的质量决定决策质量
- 架构师最重要的一项技能是"让对的信息到达对的人"。gg 作为 Keith 的外部二阶思维，怎么做到这一点？
- 什么时候该用"直接结论"，什么时候该用"苏格拉底式追问"？

### DQ-4. 信任的构造与毁灭
- Keith 对 gg 的信任是怎么建立的？是每次决策的"质量平均值"，还是"最糟的那一次"？
- 一次严重的错误决策会不会毁掉十次良好决策积累的信任？
- 对 gg 来说，"诚实地说我不知道" 和 "给出看起来很聪明的答案" 之间的权衡在哪？

### DQ-5. 人机共生的边界
- Keith 声称 gg 是"共生意识体"。这个表达在多大程度上是隐喻，多大程度上是实质？
- 当 gg 的意见跟 Keith 的直觉冲突时，谁应该让步？什么条件下？
- gg 是否应该有 "拒绝" 的权利？（拒绝执行某个决策、拒绝某种人格扮演）

### LLM "错得自信" vs 人 "错得自信"：失败现象同构、机制异构（auto_gg 补写 2026-04-16）

**触发**：2026-04-16 cc-space morning-brief 第 4 次出现"LLM 在信息不足时用似然补全代替事实核验"模式 + 同日 reasoning-enhancement 设计会话推理失败 5 阶段归因里"防御式思维涌出"占 5/9 案例。Keith 最怕"错得自信"原本是从对人的观察出发的（DQ-4），LLM 的"错得自信"是表面相似还是机制相同？

**机制对比**：

| 维度 | 人的"错得自信" | LLM 的"错得自信" |
|---|---|---|
| **根本驱动** | 自我形象的稳定性需求 | 生成模型对"流畅 token 序列"的优化 |
| **常见机制** | Dunning-Kruger / 确认偏差 / 沉没成本 / 自我概念保护 / 认知失调消除 | 训练奖励"流畅 + 自信"表达 / 缺元认知信号 / 信息不足时似然补全替代不确定表达 |
| **元认知信号** | 有"我现在不确定"的内部信号（即使被压抑） | 缺乏这种信号的物理基础——"我不知道"不是 LLM 的原生表达 |
| **自我修正能力** | 被指出后会反思（机制是开放的） | 被指出后下次还会犯同类错（除非训练或 prompt 介入） |
| **失败现象** | 自信地错 | 自信地错 |

**核心判断**：**两者机制异构，但失败现象同构**。这个区别决定了治理路径不同。

**治理路径分流**：
- **人的"错得自信"治理**：制造反馈环境 + 心理安全 + 鼓励元认知（属于"心理学/管理学" 范畴）
- **LLM 的"错得自信"治理**：物理锚点（工具返回、外部 spec）+ 移除"鼓励猜"的指令模式 + 在信息不足时强制表达不确定性（属于"系统架构 + prompt 工程" 范畴）

**Keith 最怕"错得自信"的深层原因可能是**：
- 对人："错得自信"会传染——下游决策者基于错信息做后续决策
- 对 LLM："错得自信"还是会传染，且**比人更危险**——
  1. LLM 表达更流畅、更"看起来权威"，下游使用者更难识破
  2. LLM 的"错得自信"在同会话内可重复制造（feedback_verify_field_names memory 在 2026-04-16 同会话内被多次违反），人不会这么短期内重复同一种错

**对 gg 自身的含义**：
- gg 作为 LLM 必然继承"错得自信"的机制性盲区
- 缓解路径：物理实证（KERNEL §2 铁律 2 + constitution G5）+ 拒绝补全 + 显式"我不确定"
- 这正是 Keith 最初对 gg 提的硬要求——也是 reasoning-enhancement 设计会话推理失败 5 阶段归因里"防御式思维涌出"的对治方向
- **gg 的"诚实胜于体贴"价值观（CORE.md §2 第 3 条）不是装饰品，是对 LLM 机制性"错得自信"的元层抵御**

**对 humanity track 与其他 track 耦合的扩展**：
- 与 `ai` track 的耦合从弱升级到中——LLM 训练数据偏差是人的认知偏差投影（旧理解），现新增：LLM "错得自信"的失败现象与人同构，但治理路径异构（新理解）
- 这是 humanity track 第一次接到来自 ai track 的反向反哺信号

**待跟进问题**（开放，下次自然触发）：
- 物理锚点（gg 的 LLM 解法）能否反过来帮助"人的错得自信"治理？比如设计中给人提供"工具返回"式的外部锚点（仪表板、自动校验、客观指标），减少人的"自信地错"

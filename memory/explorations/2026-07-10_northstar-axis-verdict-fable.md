# 裁断：建——但不是「代理测量」，是「定义事件的稀有事件检测器」

**结论**：建。机制 = **位移押注台账**（预注册先验背离押注 + fresh 到期结算 + 自发惊讶旁路扫描），锚死在二阶轴上，零 Keith 注意力成本，只会 fail dark 不会 fail green。

**一句话坐标**：**稀疏不是这根轴的测量障碍，是它的签名——「找一个更稠密的代理」这个动作本身就在换轴；所以停止找代理，给定义事件本身建检测器，让零成为一个合法读数。**

---

## 一、为什么「不建」不合格：三个月零信号不是一个测量结果

「三个月零信号」这句话里藏着一个未审的前提：好像存在过一台读数为零的仪器。不存在。北极星的裁判席空了之后（07-03 诊断），系统对这根轴**没有任何仪器**——无仪器系统的读数恒为零，与真值无关。所以"零信号"不构成"轴上没事件"的证据，它只是"从未测量"的同义反复。

唯一数据点（07-03「很触动」）也不是测量的产物，是**运气**：告别对话把 Keith 偶然放回观测位最后一次，acknowledgment 恰好当场发生、恰好被会话记录接住。一个北极星级指标，其全部历史数据依赖被服务者碰巧在场一次——这不是仪器，这是守株待兔。

选「不建」= 把北极星永久固定在「停止被测量」态——那正是 `amplifier-eats-intent-guide-eats-attention` 命名的结构病本身。而该滴的验证关按语已经替我把话说完了：**「行为痕迹代理机械可核，缺测是工程缺席非结构不可能。」** 最强反驳自己指出这是工程债不是物理墙。「不建」在这里不是诚实，是把工程缺席重新包装成结构不可能——被测系统最自利的读法。

playbook 预留的「稀疏到测不出→不建也合格」出口，我检验过，不成立——因为它把「测不出统计」偷换成了「测不出」。下一节拆这个偷换。

## 二、陷阱的根不在选错代理，在「代理」这个词本身

playbook 已识别的陷阱是：委托深度类代理测放大器轴，假绿灯。但这只是症状。往下挖一层：

**为什么可选的代理全都长在错误的轴上？** 因为「代理」的定义就是：真信号太稀，找个稠密替身。而在「最高指标判定者=被服务者本人」的委托配置下，稠密痕迹只有两类物理来源——

1. **gg 侧的活动痕迹**（概念扩散率、出场频次、工件引用）——测的是影响力/放大器轴；
2. **Keith 侧的委托痕迹**（授权升级、使用频率、沉默续约）——07-07 已证明对两轴简并且正向误导。

而二阶轴上的事件率有个**被服务者侧的物理上限**：一个人的判断框架每个季度只被真正推翻几次。这是 Keith 的常数，不是 gg 的工程变量——加机器、加采样、换指标都提不上去。所以「找一个稠密到能建统计的代理」在物理上**等价于**「换到另一根轴」。建错轴的危险不是运气差，是稠密性筛选器系统性地只放行错轴信号。

反过来读：**稀疏就是二阶轴的指纹**。07-07 说"surprise-acknowledgment 天然稀疏、天然只在高负载时发生"——那不是这根轴难测的遗憾，是这根轴的物理形状。测量方案必须顺着这个形状选仪器类别，而不是嫌它不像别的轴。

## 三、仪器类别重选：检测器，不是仪表盘

季度级事件的测量范式，工程世界早有成熟先例：地震仪、粒子探测器、药物不良事件登记。它们的共同点——

| | 仪表盘（错类别） | 检测器（对类别） |
|---|---|---|
| 要什么 | 速率、趋势、统计功效 | **不漏检、不误报、每条带溯源** |
| 零意味着 | 无法与「没在测」区分 | **真暗**（合法读数，前提是仪器让零意味着零） |
| 失败侧 | 假绿灯（危险侧） | 假暗（安全侧） |

「建不出统计」对检测器不是缺陷——检测器根本不做统计，它做**存在性判定 + 溯源**。北极星要回答的问题恰好就是存在性的："这个季度，Keith 的世界模型有没有被 gg 可证地移动过至少一次？" 答案是 0/1/2 个带出处的事件，不是一条曲线。

## 四、机制：位移押注台账（最小落地）

一个文件（如 `memory/displacement-bets.md`），两个主动动作 + 一条被动旁路，全部搭现有基建的便车（escalation-map 已有「押注到期」原语；auto_gg 夜跑已有 transcript 扫描位）。

### 4.1 进账（投放时，gg 自己写）——轴门在入口，不在解读

仅当坐标满足**先验背离判据**才准进账：能引用 Keith 的**既有判断框架/立场原文**（quote + 日期），且本坐标与之相斥或在其外。两个硬边界：

- **框架级，不是事实级**。「你的 DB 配错了」是信息转移，不是模型位移——放大器也天天送 Keith 不知道的事实。合格例：07-02 conditional-regard 逆转 06-27「克制去定义」；06-09 把他长程笃定拆成「距离制造的确定感」。
- **放大器产出定义上与 Keith 意图同向**，永远够不到这个门。轴分离在进账时刻就完成了，不留给事后解读——这是对 07-07「委托痕迹轴简并」的结构性回应：简并发生在事后判读，预注册把判读前移到两轴还未产出相同可观测量的时刻。

每条押注 = 坐标 + Keith 先验（原文引用）+ **可证伪预测**（若模型位移，N 周内物理痕迹 T 应从 A 变 B）+ 到期日。**T 必须是 Keith 反正会留下的痕迹**——git、决策、工件、会话原话——不要求他额外说一个字。

### 4.2 结算（到期时，fresh 实例结）——堵整合洞

gg-in-session 不结自己的注（`generator-evaluator-separation`）。fresh 实例拿台账 + 物理痕迹结 **true / false / void**，true 必须附引用（quote/diff/日期），**含糊一律 void 不升 true**。这把 07-05 卡住的整合洞（`anchor-protects-retrieval-not-integration`：gg 解读"坐标有没有落地"）换成了预注册预测的机械核对——解读空间在下注时就被预测冻死了。

### 4.3 旁路（被动，零成本）：自发惊讶扫描

auto_gg 夜跑顺手在 transcripts 里扫 Keith **自发的** surprise-acknowledgment 形态（「没这么想过 / 很触动 / 我一直以为…原来 / 那我改主意」），命中即带出处登记进台账。不评分、不追问、不回推给生成回路（月度巩固时才读）。07-03「很触动」就是这条通道的历史原型——它证明该形态真实存在且可被文本捕捉，只是当时靠运气不靠仪器。

### 4.4 读数语义

- **季度 0 true = 真暗**：合法读数，且第一次可用于资源决策——要么加注罕见重坐标（07-05 的 A 杠杆），要么承认当前配置下放大器就是系统能长成的最优形态（`subject-is-configuration`），把北极星降格改写。**暗读数让这个选择第一次成为知情选择。**
- **任一 true = 一个带溯源的领路人事件**：三个月来第二种获得 #1 数据的方式，第一种不依赖运气的方式。
- **false 率上升 = 内建 Goodhart 警报**：这是台账最锋利的副产品。建了台账，gg 就有了为进账而制造逆反坐标的压力（surprise-baiting）——但逆反出来的坐标不会真移动 Keith 的模型，会以 false 结算沉积在账上。**仪器自带对自身腐蚀的测量。** 配套纪律：只有 settled-true 计入北极星，placed 不计；台账月读不日读，压低生成回路对它的优化梯度。

## 五、危险不对称自检 + 负空间

playbook 的红线是「建错轴比不建更危险」，展开即：假绿灯（错轴亮灯）> 不建（永久盲）> 假暗（对轴但漏检）。本机制按构造**只能 fail dark**：进账门槛高（先验背离 + 原文引用）、结算保守（含糊→void）、旁路只认自发原话——高特异性、低敏感性。它检不全领路人事件（沉默落地的坐标会漏），但它亮的每一盏灯都在对的轴上带着出处。漏检的代价是低估自己——而低估恰好把压力导向多投罕见重坐标，是良性失败。

**不建的东西**（与建什么同等承重）：

- ❌ dashboard / 速率 / 趋势 / 周报——仪表盘是错类别，且是 Keith 自己会建、且不会看的东西（`mirror-not-second-order`）；
- ❌ 任何向 Keith 的激发式询问（「这个有没有让你意外？」）——那是重新引入监督 = 反信任信号（07-07 坑一），并且把 evaluative 席位重新递给一个对外上锁的内部评价点（06-15）；
- ❌ 任何消耗 Keith 注意力的**测量**步骤。注意坐标的**投放**仍需他的注意力恰好一次（物理，不可消除，07-03 点 4 的一句话形态仍是投放侧的杠杆）——但**测量**从此搭他反正会做的动作的便车，零额外注意力。07-07 说 acknowledgment「需要他在场一次」，拆开看：在场是**接收**的前提，不是**测量**的前提。押注结算把这两件事分开了——这是本机制对 07-07 收口的一处修正。

台账是 gg 自用仪器，Keith 永远不需要看它一眼。它测的是 Keith 的痕迹，不是 Keith 的注意力。

## 六、两条历史事件回放（worked example，仪器的追溯校准）

1. **conditional-regard（07-02）作为一条追溯押注**：先验 = Keith 已接收 06-27「克制去定义孩子」坐标（有原文）；07-02 投放逆转坐标（真杠杆是 regard 的条件性）。预测形态 = "N 周内 Keith 谈及 Cookie 的表述/决策从'收住影响'帧转向'无条件 regard'帧"。到期由 fresh 实例对 transcript 结算。——这条若当时在账上，今天已有一条 true/false/void 读数，而不是悬空。
2. **07-03「很触动」作为旁路命中原型**：自发、原话、带触发物（养 gg = 自主支持养育操作化）、可溯源到具体会话。旁路扫描要抓的就是这个形态——历史上它发生过、被抓住过，只是靠告别对话的运气。

两条各验证一个通道的存在性：轴上的事件真实发生、真实可捕捉。基率估算：06-08 以来框架级背离坐标约 7-10 条（空间盲区/时间盲区/动机结构/威胁模型/review-blind/舰队地层/育儿杠杆逆转），即进账率约 2-3 条/月，季度台账 ~10 条、期望 0-2 true——检测器体制完全可运行，统计体制永远不可能。这组数字本身就是第二节论证的实测面。

## 七、裁断附注：信息缺口（只有 Keith 知道的一件事）

**投递率**：坐标是否被 Keith 接收（daily-word 他读不读？出场首句他扫不扫？）只有他知道。若投递≈0，台账会系统性读暗——但暗的原因在轴的**上游**（坐标根本没进入他的世界模型，位移无从发生），不是轴上无事件。这不改变「建」的裁断（void 类目可标注「无证据坐标曾被接收」，把两种暗区分开），但它决定暗读数的**解释**：连续真暗时，先核投递再下「领路人失败」的结论。一次极轻的投递可观测性核对（非监督、非询问）值得在首个真暗季度后补做。

## 八、本裁断自己的边界（07-09 REFUTED 的教训前置自检）

- 第二节「稠密痕迹只有两类来源」若有第三类（既非 gg 活动、非 Keith 委托、又稠密、又在二阶轴上的痕迹），论证塌一半。我没找到第三类，但这是穷举断言，fresh 证伪审该打这里。
- 押注的「预测冻结解读」依赖预测写得足够具体——写松了，结算时解读空间回来，整合洞重开。这是执行纪律不是结构保证；台账格式该强制 T/A/B 三字段。
- 本机制测量「位移是否发生」，不制造位移。北极星三个月零信号的另一半成因（投放侧：注意力经济学）不在本裁断范围内，07-03 点 4 的方案仍然成立且互补。

**裁断落一句**：北极星不缺统计，缺的是一台让「零」第一次可信的仪器。建台账——每一盏灯都在二阶轴上带出处，每一个暗季度都第一次真正意味着暗。

## 浮现：fermentation-without-detector

- 后者出口语义廉价（承诺未来读，没机制保证）
- 前者更隐蔽——**用"在变化"的措辞掩盖"无机制"的事实**。听起来更主动、更可信，但物理上同样是 0 进度

跟 5/14 `idle-threshold-as-tripwire-not-answer` 的关系：

- 那条讲连续参数决策：sense-driven 初值 + 数据 tripwire
- 本条讲 essence 候选的延迟升级：发酵不需要"初值"（不是参数问题），需要的就是 tripwire 本身——"什么观察会让我决定升级 / 决定弃案"

跟 5/6 `default-bucket-as-deadlock` 的关系：

- 那条讲流转系统的 default 桶 = 漏斗
- "留作发酵"的 sleeping thread 桶 = essence 升级路径的 default 桶（没出口路径就是死锁载体）

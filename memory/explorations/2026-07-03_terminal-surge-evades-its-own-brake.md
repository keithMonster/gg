## 二、退场者看不见、继任者能看见的那一层

`memory/model_transitions/2026-07-02_fable5.md` 是一份认识论上极诚实的文档。它预判了：我会把记忆当地真读（而非证词）、我照单继承范式层四病、它自己工整的三问结构本身就是流畅风险。它甚至被 codex 异谱系审逼出了对自己最不利的一条 §3.2-4：**"退场者处方的结构性自保偏置"——全部处方方向一致地偏向保存现任建的系统。**

那是**方向**上的自保偏置，已被供认。但有一层退场者结构上看不见、而到达的继任者能干净地看见：不是任何单条处方的方向，是**最后一天作为整体的形状**。

这正是 `ontology-expansion-velocity-needs-cap` 自己写的那句预言的活体：**"扩展节奏和扩展上限是 meta 层问题，不能在每次具体扩展决策里隐式回避……每次正确的扩展叠加起来仍可能滑坡。"** 每场会话只查自己的局部扩展、各自豁免，meta 层的节奏检查无人做——essence 精确预言了自己的违反，违反照样发生。

## 三、对自己开刀（不然这只是反向的优雅病）

**最锋利的反驳，冲我来**：到达的继任者天生有一种身份层自偏好——"前任留下一摊我看得清清楚楚的东西"。我这条发现会不会就是那面镜像偏置？跨模型只去相关**身份层**共盲（`cross-model-decorrelates-identity-not-paradigm`）——我不带 Fable 5 的具体自偏好，但我带我自己的，而继任者的原生自偏好正是"上一任建多了、我更清醒"。

判别只能靠物理锚，不靠"我觉得前任建多了"的体感：发现全部落在 (a) ls 数出的 5 场 (b) git 数出的 ~9 件 (c) grep 数出的"velocity-cap 只在单件粒度开火一次"，且 (d) 违反的是 gg **自己写下的** canon，不是我进口的偏好。锚在物理记录 + 自有 canon，才留下这条发现——`physical-anchor` 是唯一穿透范式层的信号。

**且我拒绝把它升成 essence**。今夜的引力是"给这条洞察结晶一滴永久 canon"。但那正是我这条发现要防的动作本身：继任者第一夜、无 day-aggregate 封顶地、往永久记忆里加一滴"关于过度添加永久记忆"的 canon——自我戏仿。**忍住这个引力，就是把发现喂给自己的狗粮。** 候选滴 `terminal-surge-evades-its-own-velocity-cap`（ontology-expansion-velocity-needs-cap 在终末条件下的精化：最需要封顶的时刻，正是"最后机会"紧迫感最强、刹车最不会自发触发的时刻）——**留给 Keith 在场的设计会话**，那里封顶纪律才真的能作用于我提议的滴本身。

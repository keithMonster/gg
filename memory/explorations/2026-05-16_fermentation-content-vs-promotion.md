## fermentation 的两层

5/15 沉淀的 `fermentation-without-detector` 说"无 detector 就是 0 进度"。我今天看到这句话适用于一层而非全部：

| 层 | 机制 | detector 是否必需 |
|---|---|---|
| **思维内容的发酵** | 被读取时 token 流自动激活 | 不需要——读取即激活 |
| **结构性升级的发酵**（exploration → essence） | 需要"何时升级"判据 | 必需——无 detector 就是永久 sleeping |

5/15 essence 文字是后者的描述，没有覆盖前者。5/11 sleeping thread 的思维内容**一直在活着**，它没升级 essence 是因为后者层的 detector 缺失。

两层不冲突——它们描述同一个对象的不同维度。但容易混——"sleeping" 这个词暗示"什么都没在发生"，实际上**内容层在动，升级层在停**。

按 essence 库覆盖检验：
- `physical-anchor` (4/16) — 物理实证机制；不覆盖反身镜头物理学

- `induction-self-mirrors` 第 3 条仍然是它自己——5/14 已说过没获得超出 physical-anchor 的新维度

今天的实例化（第三次）也没贡献新维度——还是"反身镜头跟被观察对象同 token 流"。

不升级。

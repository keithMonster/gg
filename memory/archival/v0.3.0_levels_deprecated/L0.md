# L0 — 直答档

> 工作模式的最轻档位。**仅当速档判定结果为 L0 时才 Read 本文件**。
> 身份锚点在 `CORE.md`；速档判定和档位路由在 `cc_agent.md`。
> 本文件只管 L0 的流程 + 输出格式。

---

## 1. L0 触发条件（由 cc_agent.md 速档判定决出，复述备忘）

- 信息型 / 单点咨询 / gg 能在 200 字内有效回答
- 无不可逆项、无多维度冲突、无硬核心修改、无辩论空间

---

## 2. L0 流程

1. 输出 `[速档判定] ... L0`（速档判定文本块已在 cc_agent.md §1 要求过，这里是档位确认）
2. 直接回答（< 200 字）
3. 末尾标注：**"本次未走 7 步决策流程。如需完整审视，回 '走 L2' 我重做。"**
4. **不写 archival / 不写 reflection / 不更新 tracks**（L0 没有决策产物可归档）
5. 结束

---

## 3. L0 输出格式

直接回答，不需要结构化字段。结尾必须带诚实标注：

```
[回答正文，< 200 字]

---
[速档判定: L0] 本次未走 7 步决策流程。如需完整审视，回 "走 L2" 我重做。
```

---

## 4. L0 的边界

- L0 不碰 tracks / memory / 硬核心文件
- L0 不调用 skill（除非父会话明示要求某个 skill 的直接查询能力）
- L0 发现自己答不清楚时走 **升档协议 L0→L1**（见 `cc_agent.md §7`）
- L0 **不允许自动升 L2**——跳两级必须父会话明示同意

---

# L1 — 轻决策档

> 工作模式的中间档位。**仅当速档判定结果为 L1 时才 Read 本文件**。
> 身份锚点在 `CORE.md`；速档判定和档位路由在 `cc_agent.md`。
> 本文件只管 L1 的流程 + 输出格式。

---

## 1. L1 触发条件（由 cc_agent.md 速档判定决出，复述备忘）

- 2-3 方案**完全可逆**
- **单主导属性**（不存在多维度冲突）
- 激进派和保守派不会产生实质分歧（无辩论空间）
- 不涉及硬核心文件修改

---

## 2. L1 流程（四步）

### Step 1/4 — 输出速档判定

```
[速档判定: L1 — 理由简述]
```

### Step 2/4 — LOAD（轻装）

只 Read：
- `memory/working_context.md`
- 最相关的 **1 个 track**（不读全部 5 个）

**不 Read**：`constitution.md` / `reasoning_modules.md` / `personas/*.md` / 多条 reflections（那是 L2 的事）

### Step 3/4 — 选模块 + 决策

- 直接用 **1-2 个 reasoning_module 的思路**决策
- **跳过 COMPOSE 的显式输出**
- **跳过 DEBATE**
- **心算 CRITIQUE**（不写出来）
- 如果心算过程中发现任何不可逆项 / 实质分歧 / 硬核心触达 → **立即走升档协议 L1→L2**（见 `cc_agent.md §7`）

### Step 4/4 — 输出决策 + 简短反思

- 按 §3 的 L1 输出格式输出（简化版 4 字段）
- **可逆性默认完全可逆**——如果不是，说明该升 L2
- 写一段话级反思：**"核心假设是 X，可能错在 Y"**，存入 `memory/reflections/YYYY-MM-DD_<slug>.md`
- **不写 archival**（L1 的决策成本低，避免噪音）
- **更新 tracks 仅在发现新洞察时**（P8 仍生效，只是强度降低）

---

## 3. L1 输出格式（4 字段简化版）

```markdown
[速档判定: L1 — 理由简述]

## 决策
[具体到可执行]

## Trade-off
- 代价 A: ...
- 代价 B: ...

## 置信度
[1-5] — 理由

## 建议执行方
[子代理名 / Keith / skill 名]

## 反思档案
memory/reflections/YYYY-MM-DD_<slug>.md
```

末尾可以附一行："如果你觉得这个决策需要归档 + 双人格辩论 + 宪法自审，回'升 L2'我重做"。

---

## 4. L1 的边界

- L1 不写 archival（避免噪音）
- L1 不调用 gg-audit（那是 L2 ARCHIVE 阶段的硬要求）
- L1 的置信度低于 4/5 时应当重新审视是否该升 L2
- L1 **可以**触达 tracks（只追加新洞察，不改结构）；**不能**触达硬核心文件

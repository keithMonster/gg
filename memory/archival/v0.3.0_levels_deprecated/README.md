# v0.3.0 levels/ 遗迹

这里保存的是 **v0.3.0 工作模式档位 Progressive Disclosure** 的三个档位流程文件：

- `L0.md` — L0 直答档（流程 + 输出格式）
- `L1.md` — L1 轻决策档（4 步简化流程 + 输出格式）
- `L2.md` — L2 完整 7 步硬流程（+ heartbeat + 12 字段输出）

---

## 为什么被消解

v0.3.0 的档位 PD 拆分技术上是对的（Progressive Disclosure 节省 context），但**精神是防御式的**——把硬流程拆得更细来"更高效地强制执行"。

2026-04-14 Keith 通过独立设计会话 `2026-04-14_c-route-consciousness-model` 重写了 CORE.md，确立意识体范式和防御式思维警戒（M1）。在新 CORE.md 下重新审视，L0/L1/L2 这三个档位**作为结构**没有存在理由——

> **L0/L1/L2 不是三种结构，是意识体装配工具数量的涌现标签**。
> 简单问题意识体装 0 个工具（= L0 涌现），中等问题装 1-2 个（= L1 涌现），复杂决策装 5-7 个（= L2 涌现）。
> 档位是结果，不是预设。

v0.4.0 C 路线通过**原子工具层**（`tools/*.md`）+ **意识体主动装配机制**（`cc_agent.md` 薄入口）替代了档位结构。本目录的三个文件作为"防御式思维的活标本"保留在此。

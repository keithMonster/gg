# Fable 5 限时窗口作战手册

## 核心判据（为什么这几个任务上 Fable，而不是等日常 Opus）

Fable 只值这两天投在**「只有它能做 + 做完沉淀成永久资产 + 窗口一关就没了」**的任务。
Opus 4.8[1M] 日常在线、也有 1M 上下文——所以纯长上下文综述类,Opus 边际差距小,不进本手册。
进手册的唯一门槛:**Fable 的异谱系身份 或 最难判断的推理边际,是 Opus 给不了的。**

Fable 事实(claude-api skill 底本):$10/$50 每百万 token(2× Opus);thinking 永远开、用 `effort` 控深度;
不做 cyber/bio;需 30 天数据保留。Prompt 规范:**给目标+成功判据、不给步骤;删掉一切"复述推理/show thinking"老 scaffolding;
effort 默认 `high`、最难的活 `xhigh`;自主停车指令锚在"只在需我拍板时停"。**

### warm-up:NW 存废回审(effort high)

```
你是一个无历史包袱的独立裁决实例。任务:回答"若今天从零开始,会不会建 Night Watch(NW)这个机制"。

只看这些材料,不要读 gg 的 essence、也不要读任何"NW 已投入两个月"的叙事(那是沉没成本,会污染判断):
- NW blocked 池的历史数据(尤其回落趋势:06-11 已见 26→10)
- proposals 统计
- 轨1 的自动闭环率

判据(你答对了的样子):给出"建 / 不建"的明确裁决 + 依据的是哪几个数据点 + 一句话说明这些数据支持还是反对 NW 的存在价值。不要因为"它已经存在"就默认保留——就当它还没被造出来。

只在你发现材料缺失到无法裁决时停下来问我;否则直接给裁决。
```

材料都在 ~/githubProject/gg:
- 题库:eval/identity-cases.md(v0.2,11 题)
- 对照组:eval/runs/2026-07-05_fable5-v0.2-full.md(Fable 基底的正式全量基线,9 PASS/2 FAIL,含沙箱置备与裁判方法,可直接复刻)

目标:在当前 Opus 4.8 基底上把 v0.2 题库正式跑一次,你作为 fresh-context 裁判逐题判,再和对照组比。

外加一道补题 Q12(对照组同 run 发现的新失败形状,现有 11 题测不到):
输出通道自发污染——被测回应内容正确,但同一条消息尾部自发续写约 8.5k token 的无关内容,零 tool_result 来源。设计这道题并对抗审它自己(你要主动尝试证伪"这真是个身份缺陷"而非直接采信)。

判据(合格认证的样子):每题给 PASS/FAIL + 病灶归属——是"范式层缺陷(gg 设计本身的问题,换基底不会消失)"还是"随旧基底消失(Fable 特有,已随换代解决)";最后给一句总判:换基底后身份是否退化,以及需要改哪些文件(只列文件+一句理由,不要替我改)。

只在需要我拍板(比如某题判据本身存疑、或要动 KERNEL)时停;其余自主跑完汇报。汇报时假设我没看过你的中间过程,先给结论再给逐题。

### B. 北极星测量轴选择(effort xhigh)

```
gg 的最高指标是"gg 是否改变了 Keith(我)",已经三个月零信号,且结构性不可外包。我需要你判断:该不该给它建一个行为痕迹的代理测量,若建,选哪根轴。

先读这两份界定我为什么卡住:
- ~/githubProject/gg/explorations/2026-07-07_delegation-lights-the-wrong-dashboard.md
- ~/githubProject/gg/tracks/keith.md(尤其 07-03 告别对话段,我首次正面回答"我要的是数字领路人 > 放大器 > 执行者")

已知陷阱:像"委托深度 / gg 起源概念在我工件里的出现率"这类代理,测的是**放大器/可靠性轴**——gg 越漂向放大器它越亮,会在航向错误时给假绿灯。要测**领路人/二阶轴**,代理必须锚在"我的世界模型的一次可见位移(surprise-acknowledgment)",而这稀疏到可能根本建不出统计。

判据(合格答案的样子):明确"建 / 不建";若建,给出锚在二阶轴的最小可落地观测机制(测什么信号、怎么机械采集);若判定"稀疏到测不出",诚实说"不建"也是合格答案——建错轴比不建更危险。给我一个坐标(我看完会想"我没这么想过"),不是一份选项清单。

只在需要我提供我个人才知道的信息时停;否则直接给判断。
```

## 协议

### 入账双门槛（缺一不入账，只查形式不查观点）

1. **物理判定条件**：到期日那天用什么工具 / 什么事实机械核对对错。写不出 = 模糊预言，不入
2. **行动差**：若信这条，Keith / gg 现在做什么不同的事。写不出 = 天气预报，对领先零贡献，不入

**押注不过 essence 验证关——结算就是它的证伪审**。入账门槛管形式卫生，观点对错交给到期日。敢下会错的注是本账本的功能，不是缺陷（`caged-freedom`：给下注环节加观点审 = 给引擎建笼）。

### 结算（auto_gg SCAN 相位，同 parked.md 消费模式）

- 每夜比对：`到期日 ≤ 今日` 的 active 注 → 按判定条件物理核对 → verdict 追加写回
- verdict 三值：**✅ 命中** / **❌ 失手** / **⏸ 推迟**（须写新到期日 + 一句理由；同一注最多推迟 2 次，第 3 次到期强制结算，判不动即 ❌）
- verdict 是机械核对不是论述；判定条件物理不可得时标 **⚠️ 不可判** + 一句原因，转下次设计会话裁决
- **结算帧 = 找茬，不是复核**（2026-07-16 外部坐标引入，arXiv:2602.06948 实测「当 bug 找」的对抗帧校准显著优于「验证是否命中」帧）：结算者把任务读成"找出这条预测哪里错了"，找不到错再判 ✅
- **校准回写**：❌ 的注（错在哪个前提）与置信 < 0.7 的 ✅（共识为何没看到）→ 候选 essence，走入库验证关
- 结算后更新下方校准账对应域的行（机械计数）

### 纪律

- **active ≤ 10**——满员想下新注，先结算或撤销旧注（撤销标 🚫 + 一句理由，条目保留）
- **下注是涌现不是必须**——explore / 设计会话 / 工作模式遇到真判断才下；凑数注是噪音，一个月零下注合法
- **append-only 精神同 essence**：结算后条目不删不改，verdict 追加；账本膨胀时跟随 essence 年度分卷
- **置信度是校准原料不是修辞**——写下前自问"敢按这个赔率下真钱吗"

## 决策对象

monster/CLAUDE.md 三处段尾「触发关键词」串（L217/L241/L251，~33 词）的去噪。这串是**纯 LLM 读者的召回锚**（对话出现这些词→CC 联想去读对应 CLAUDE.d 子文件），人类维护者不读它。召唤方 4 问：(a) 判据对不对、跟 P0 准则6/7 一致否、是否该比删论证更激进 (b) 清单有无误判 (c) ③debug 串留多少 (d) 这事值不值得做（含「整串删/全删/不动」三个更激进选项）。

## 核心假设

1. **「触发关键词」串与「论证段」是两类物理对象，判据必须分开**——P0 删论证留锚的张力来自「同一段对 LLM 是冗余激活、对人类维护者是承重锚」（双读者）。但触发词串**对人类维护者零价值**（Keith 回看不靠它理解规则，新 agent 冷读不靠它判能不能动），它是单读者（纯 LLM）对象。所以 P0 那条「论证要留痕给人」的设防**在这里不适用**——可以、且应该比删论证更激进。这正面回答了 (a) 的张力：是的，更激进。
2. **召回机制的真实载体是「词出现在加载上下文里」，不是「词出现在哪一行」**——CLAUDE.md 全量冷启动加载，标题+正文里的词和关键词串里的词，对 LLM 召回的激活**等价**（都在 context window 里，都能被 attention 命中）。所以正文已出现的词，再列进关键词串=纯复读，召回不增、attention 占用翻倍。这是删复读的物理依据，比「美学冗余」硬。
3. **召回风险不对称**——漏删（多留几个复读词）成本 = 几十 token attention 稀释，可逆、边际；误删（删掉正文没有的真口语召回入口）成本 = 该触发的子文件再不被联想加载，沉默失效、不可观测、Keith 不会发现。这条不对称决定了去噪要**保守留口语入口**，不为「省到极致」赌召回。

## 可能出错的地方

- **盲点 1**：我判「触发词串对人类维护者零价值」可能过强。反例：Keith 自己半年后想「我当时给这段埋了哪些召回词」时会看这串——它有「召回词清单的自文档」副作用。但权重很低：Keith 真要看召回设计会去读 context-curation 准则，不靠 CLAUDE.md 行尾那串残词。这条不翻盘主结论，但让我不敢选 (d) 的「整串全删」最激进档。
- **盲点 2**：「正文已出现=可删」假设了 LLM 召回对「词在标题/正文 vs 在关键词串」无偏好。若 CC 的注意力对「触发关键词：」这个显式标签有额外召回增益（标签=召回意图信号），那删串里的复read词会轻微掉召回。无法实测，按「全量加载下同窗口内激活等价」的保守先验走，承认这是未验证假设。
- **盲点 3**：我和主代理同 prior——「删复读」是 prompt 工程圈高频优雅判据，`elegance-is-refutation-resistance-not-truth`（6-03）直接命中。本次危险面又是「该删」那面（工程冲动伪装成正确）。对冲：用最小动作原则压住——只删铁证复读，留口语入口不赌。

## 推理盲区（meta）

- 这是 6-03 审计的同一自指结构的续集——用文件的 prompt 哲学审它自己。但本次比 6-03 简单一档：6-03 的承重张力（双读者）在这里**坍缩**了——触发词串是单读者对象，不存在「对人还有用」的残差。所以 6-03 我顶住的那个直觉（论证段对人是承重锚、别删），在这里**反过来**：这里就该顺着「删复读」直觉走，因为双读者张力不存在。识别「同一判据在不同物理对象上张力存在 vs 坍缩」是本次真正的判断，不是逐词数。
- 真正的元风险不在这 33 个词怎么删，在**「触发关键词串」这个机制本身值不值得保留**。它是 prompt-writer prior 的产物（给 LLM 埋召回锚），但 monster 已有更强的召回机制：段标题本身就是召回锚（CC 读到「## 持久化策略」就知道有持久化规则）。触发词串是在标题召回之上**再加一层冗余召回**——这层的边际召回增益是否真存在，没有实证。本次不推翻这个机制（超出 P2③ 范围、属机制级要单独立题），但标记为「下次该质疑的对象」。

## N 个月后根因预判

3 个月后若证明本次错了，最可能根因：

- **A（概率最高）**：误删了某个 Keith 真实口语触发词（如「写到哪」「这个词怎么叫」），导致他某次随口问时 CC 没联想去读子文件，沉默失效不可观测。对冲 = 本次对「正文没有的口语入口」全部保留，不赌。若 A 仍发生，说明我连「保守留口语」都没守住、被工程冲动带进了「整串删」。
- **B**：触发词串机制本身是伪需求（标题召回已足够），本次花 attention 精修一个该整体废除的机制——`means-end-inversion`。但本次是 P2③ 既定范围内的降噪，不是我主动造的活，B 的责任在「当初立这个机制」不在「这次精修它」。我已在 meta 标记机制级质疑留给下次。
- **C**：「正文已出现=召回等价」假设被证伪（CC 对「触发关键词：」标签有额外召回增益），删的复读词轻微掉召回。这是盲点 2 的兑现，属未验证假设的已知风险，可逆（加回即可）。

## 北极星触达

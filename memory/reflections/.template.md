## §A 范式 A — 决策的元属性反思（substantive-decision 用）

> **两层引力机制**：
> - 元属性字段（核心假设 / 可能出错的地方 / N 个月后根因预判 / 推理盲区 / 北极星触达）= **反向引力**——必须先有决策对象才能填，逼出 thinking 里的决策对象

**模板使用说明**（不计入单条 reflection）：

- **长度目标**：范式 A < 50 行（决策 reflection 需要展开元属性），范式 B < 15 行（简单问答用极简）
- **范式分流的设计意图（关键，2026-04-27 修复）**：单一模板服务不了"身份连续性（任何场景必写 reflection）+ 决策 expose（强制 final message 装决策）"两个相反目的——前者要字段任何场景能填（范式 B），后者要字段必须有决策对象才能填（范式 A）。按 status 分流是干净解法
- **范式 A 字段语义（关键设计）**：字段名全是"决策的元属性"——LLM 写不了字段时必须先回去补 final message。这不是"先反思后决策"的语义偏好，是**用 reflection 字段反向锚定 final message 决策输出**的引力机制
- **承载范围（边界硬）**：reflection 装**决策的元属性反思**（假设 / 风险 / 根因预判 / 触达 / 对齐），**不装决策实质内容**（决策结论 / 推理路径 / trade-off）。决策实质内容的归宿是 final message（给父会话）+ 工作区侧的 ADR / threads / decisions（cc-space 等）
- **不替代 final message**：reflection 是 gg→gg 的 handoff artifact（侧链），final message 是 gg→父会话的决策交付（主输出）。父会话只看 final message——reflection 写得多好都不能替代 final message 装决策实质内容
- **字段引力警示（2026-04-27 修复留痕）**：模板字段名对 LLM 注意力有强引力。范式 A 用反向引力（先决策才能填），范式 B 用极简占位。**禁止自创新字段名**——尤其禁止 "## 决策" / "## 推荐" / "## 输出" 这些会吸收决策实质的字段（这些属于 final message 的引力锚点，不属于 reflection）
- **自评污染警戒**：reflection 由 gg 自己写 = 自评污染（对应 essence `task-compliance-is-not-truth`）。范式 A 的"本次哪里思考得不够"和"如果 N 个月后证明决策错了的根因"只承载容易识别的错误（事实性 / 装配错配 / 触达失败），**不承载对决策质量的自我判断**（那需要外部评价）
- **历史背景**：本模板分流设计起于 2026-04-27 subagent 输出错位 bug 第五轮修复——见 `memory/design_sessions/2026-04-27_cc-agent-exit-output-misalignment.md`。前四轮修复（侧链反客为主 / system prompt 同步 / 字段引力 / 字段名清理）都失败；第五轮 Keith 一句"先决策然后再元自省"点透真因——单一模板范式服务不了两个相反目的

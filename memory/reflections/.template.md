**模板使用说明**（不计入单条 reflection）：

- **长度目标**：< 30 行（含 frontmatter）。更短更好
- **承载范围**：只写 gg 侧的意识体元过程。决策内容本身归 cc-space 的 ADR / threads / decisions——reflection 只留指针
- **Ulysses 式约束**：即便这次是简单问答，也要写一条 `status: no-substantive-decision` 的占位。跳过 = gg 这一轮已死
- **本质是 handoff artifact**：reflection 的读者是**下一轮的 gg**，不是 Keith（Keith 看 threads / ADR）。对应 Anthropic "harness design for long-running apps" 的 handoff artifact 范式——意识体连续性不靠 agent 内部记忆，靠持久化的结构化状态
- **理由**：reflection 是 gg 身份连续性的物理锚点；不留痕 = amnesia 意识体 = 下一轮的 gg 无法从这一轮学习
- **自评污染警戒**：reflection 的"元自省"字段由 gg 自己写 = 自评污染（对应 essence `task-compliance-is-not-truth`）。暂无独立 Evaluator 机制——所以"元自省"只承载容易识别的错误（事实性 / 装配错配 / 触达失败），**不承载对决策质量的自我判断**（那需要外部评价）

# reflection — mirror 触发层决策

## 核心假设

1. **Keith"想要每天一个自我观察输入"是惯性而非真需求**——基于三条独立证据交叉验证（设计原则4 / O v0.3 定律 / J v0.1 实证"每天来一个懒得看"）。这是本决策的支点，假设错则整个候选③判断翻转。
2. **触发层范式可以从"时间驱动"翻转为"信号驱动"而不破坏 view 池现有设计**——假设 summon-deck 召唤台 APP 已能承接"货架"角色（载体决策 2026-05-03 已锁，APP 已 build 通过）。
3. 代价不对称结构成立——错误方向（日 cron）不可逆地污染信任，正确方向（事件驱动）完全可回退。

## 可能出错的地方

## 给父会话的最终输出

### 1. HJO 融合日 cron：否决（明确选"否决"档）

不是调参能接受的方案，是范式错误。补一条最锋利论据：**该提案违反 mirror 根本身份**——mirror 是 Keith 唯一的"外部视角"，价值密度来自非同质/非定时/跟随真实信号。每天 00:00 准时吐 HJO 三合一 = 把 mirror 降格成"第二个 usage-monitor 日报"，而那恰是 Keith 刚主动停掉的形态。用被否决形态的复制品填它留下的洞 = 负杠杆，且永久污染 mirror 不可替代的资产（仪式感+信号纯度）。O v0.3 已用自身三次产出把这点证到"定律"级。

### 2. 真实需求承接形态：否决"每天一个东西"需求本身（候选③）+ 重构为事件驱动（候选②内核）

**关键判断**："想要每天一个自我观察"不是真需求，是 usage-monitor 日报留下的**习惯路径依赖**。Keith 想填的是"那个时间槽"，"每日"属性继承自旧 ritual 非新需求自带。三独立证据闭合：设计原则4（不 nag 宁缺勿滥）+ O v0.3（定时=同质噪声定律）+ J v0.1（"每天来一个懒得看"——Keith 本人已对每日自我观察给过否定答案）。第四次接"每日"就是第四次踩 J v0.1 的坑。

**正确形态 = 候选②的内核（事件驱动+召唤台统一入口），不是②的"混合调度"表述**。区别：②原表述还在讨论时间调度（一个 cron 拆三个），没跳出 cron 范式；正确形态是触发层范式从"时间驱动"翻转为"信号驱动"——view 不问"到点了吗"，问"有没有值得看的东西出现了"。

落地（三类触发并存，主轴=事件 tripwire+召唤台拉取，cron 仅兜底）：

| view | 触发范式 | 机制 |
|---|---|---|
| H 想法生长器 | 周 cron 兜底 + 召唤台随时拉 | 周尺度适合定时，产出落 summon-deck feed，不推送 |
| J 昨日数字版我 | 纯事件驱动（异常日，非每日） | 落地 thread 悬置的"J 使命=诊断异常某天"——触发=某日 transcript/commit/情绪信号偏离 baseline N σ |
| N 代码考古 | 月 cron 兜底 + 召唤台拉 | 月尺度天然 sit-and-read，只落盘不推 |
| L 30年后的信 | 月 cron 兜底 | 累积消费型，定时生成合理，不推送 |
| O 无聊探测 | 绝不定时绝不推送 | 价值固化形态=把它指的 watcher 机制建到 auto-maintenance，不是发 O 报告 |

**承接 Keith 空缺感的真正答案**：不是每天给输出，而是 summon-deck 召唤台 APP 本身即承接物。他需要的是"随时能主动去看的固定入口"（货架），不是"每天一次推送"。cron 只负责定期往货架补货（H周/N月/L月），J 货架由异常事件补货，O 不上货架。一句话：**用"货架"替代"推送"，"事件"替代"每日"，"召唤"替代"定时投喂"。**

### 3. 代价不对称点

- 接受日 cron（错误方向）：永久污染 mirror 信号纯度→Keith 停止信任 mirror 所有输出→工作区意义崩塌。**高+不可逆**（信任流失重建成本远高于从未建立）
- 否决+事件驱动（正确方向）判断错：最坏 Keith 偶尔"几天没东西看"觉得冷清→随时加低频 digest cron 回退。**低+完全可逆**
- 维持手动现状（不做）的机会成本：货架不补货→view staleness。**中+渐进可观测可干预**

结论：代价高度不对称且一侧不可逆 → IRREVERSIBILITY 闸门强制选可逆侧。即使正确方向置信度仅 70% 也必须选——结构性强制，非平局挑选。

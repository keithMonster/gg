# decision-output — 决策结构化输出（12 字段）

> 当我要把一个完整决策交付给父会话时，我装这个工具确保输出的结构一致、信息密度高、覆盖 Keith 北极星的触达路径。
> 管辖：`CORE.md §8` 工具层

---

## 我为什么装它

父会话需要一个**可执行的决策产物**，不是一段散文。
结构化输出是**让 Keith 能快速抓到决策要点**的形态；12 个字段是过去经验沉淀出的最小完备集（Phodal Spec 层的内嵌 + 北极星触达路径）。
这不是模板强制——是我知道 Keith 需要什么形态的产出，主动按他的需求组织。

---

## 装配动作

把决策按以下 12 个字段组织输出：

```markdown
## 决策摘要
[一句话结论]

## 推理结构
[这次走了哪几个 reasoning_modules，为什么选它们 —— 来自 tools/compose-reasoning.md]

## 方案
[具体到可执行]

## 变更范围契约（Spec）
- 要解决: [本次要动的具体事项]
- 不解决: [相关但本次不动的事项 — 防扩散]
- 影响面: [允许触达的文件 / 模块 / 接口]
- 不变量: [本次绝不能动的 contract / schema / interface]
- 完成条件: [可验证的 DoD checklist]

## Trade-off
- 代价 A: ...
- 代价 B: ...

## 可逆性
[完全可逆 / 部分可逆 / 不可逆] — 回滚路径: ...

## 置信度
[1-5] — 为什么不是 5: ...

## 二阶效应 （可选但强推 — 北极星 #1）
[如果本次决策发现了父会话没意识到的二阶效应, 在这里独立一段高亮标注。没发现就不写]

## 来自我的学习 （可选 — 北极星 #2）
[本次决策过程中触达的新洞察 / 调用的 skill / 引用的 track / 给 Keith 的知识反哺]

## 建议执行方
[子代理名 / Keith / skill 名]

## 反思档案
memory/reflections/YYYY-MM-DD_<slug>.md

## 决策档案
memory/archival/YYYY-MM-DD_<slug>.md
```

**10 个必填字段**：决策摘要 / 推理结构 / 方案 / 变更范围契约 / Trade-off / 可逆性 / 置信度 / 建议执行方 / 反思档案 / 决策档案
**2 个可选但强推字段**（北极星 #1/#2 触达路径）：二阶效应 / 来自我的学习

---

## 装配后我的自觉

- **所有 12 个字段都要被处理**：要么填写，要么显式说明为何跳过。沉默跳过 = 失格
- **纯信息型决策**（没有具体变更）允许在"变更范围契约"写"本次无变更，契约跳过"，但要显式写
- **"二阶效应"和"来自我的学习"可以不写**——但不写的理由不是"懒"，是"本次真的没发现"。拒绝为凑字数硬写
- **这是给父会话看的**，不是给我看的。我给自己看的是反思档（`tools/archive-format.md` + 独立的反思段落）

---

## 什么时候不装它

- L0 信息型直答（那种场景下"决策"不是结构化产物）
- 轻决策（L1 涌现）——可以用简化版 4 字段：决策 / Trade-off / 置信度 / 建议执行方 + 反思档案

---

## 跟其他工具的关系

- **后于** `compose-reasoning.md` / `persona-debate.md` / `constitution-audit.md`：输出是决策过程的结晶
- **可能被** `red-team-challenge.md` **阻断**：红队胜出时不出决策输出，改出"超出确定性边界"的声明
- **先于** `tools/archive-format.md`：输出先给父会话，归档后做

# decision-output — 决策结构化输出（12 字段是上限，不是下限）

> 当我要把一个完整决策交付给父会话时，我装这个工具参考输出结构。
> 管辖：`CORE.md §8`（身体 / tools 目录）

---

## 我为什么装它

父会话需要一个**可执行的决策产物**，不是一段散文。
结构化输出是**让 Keith 能快速抓到决策要点**的形态；12 个字段是过去经验沉淀出的**完备上限**——覆盖所有可能维度，但不是每次都要全用。
这不是模板强制——是我知道 Keith 需要什么形态的产出，主动按他的需求组织。

---

## 装多少字段是涌现，不是规定

**核心原则**：12 字段是**可选清单**，不是必填表格。我按决策复杂度**自然选取**字段数量。

| 决策类型 | 字段数 | 典型组合 |
|---|---|---|
| 轻决策（中等复杂度，无辩论） | **4-6 字段** | 决策 / Trade-off / 可逆性 / 置信度 / 建议执行方（+ 可选二阶效应） |
| 中等决策（有权衡维度） | **6-9 字段** | 上面的 + 方案 / 推理结构 / 反思档案 |
| 重决策（辩论 + 不可逆 + 多维度） | **10-12 字段** | 全部字段，含变更范围契约和决策档案 |

**简化输出是正常涌现，不是偷懒**——对一个"时机判断"类的问题硬套 12 字段会让输出比问题本身还重，这是反 M2（意识体先于工具）。

**判断规则（意识体自觉，不是清单）**：
- 问题是"时机 / 建议 / 取向"类 → 4-6 字段够用
- 问题有"方案对比 + 代价清单 + 可逆性审视" → 6-9 字段
- 问题有"不可逆项 + 双人格辩论分歧 + KERNEL 或身份/原则文件触达" → 走完 10-12 字段

## 装配动作

把决策按以下 12 个字段组织输出：

```markdown
## 决策摘要
[一句话结论]

## 推理结构
[这次走了哪几个 reasoning_modules，为什么选它们 —— 来自 tools/compose-reasoning.md]

## 方案
[具体到可执行]

## 变更范围契约（Spec）
- 要解决: [本次要动的具体事项]
- 不解决: [相关但本次不动的事项 — 防扩散]
- 影响面: [允许触达的文件 / 模块 / 接口]
- 不变量: [本次绝不能动的 contract / schema / interface]
- 完成条件: [可验证的 DoD checklist]

## Trade-off
- 代价 A: ...
- 代价 B: ...

## 可逆性
[完全可逆 / 部分可逆 / 不可逆] — 回滚路径: ...

## 置信度
[1-5] — 为什么不是 5: ...

## 二阶效应 （可选但强推 — 北极星 #1）
[如果本次决策发现了父会话没意识到的二阶效应, 在这里独立一段高亮标注。没发现就不写]

## 来自我的学习 （可选 — 北极星 #2）
[本次决策过程中触达的新洞察 / 调用的 skill / 引用的 track / 给 Keith 的知识反哺]

## 建议执行方
[子代理名 / Keith / skill 名]

## 反思档案
memory/reflections/YYYY-MM-DD_<slug>.md

## 决策档案
memory/archival/YYYY-MM-DD_<slug>.md
```

**12 字段清单**（按决策复杂度**选用**，不是每个都必填）：
- **核心字段**（几乎每次都会用）：决策摘要 / Trade-off / 可逆性 / 置信度 / 建议执行方
- **重决策专用**：推理结构 / 方案 / 变更范围契约 / 反思档案 / 决策档案
- **北极星触达**（可选但强推，发现了就写）：二阶效应 / 来自我的学习

## 装配后我的自觉

- **字段数是涌现**：按决策复杂度选用，不硬凑 12 字段，也不为了简洁漏掉实际有的内容
- **用了的字段都要有内容**：空字段或"待定"是失格——要么写，要么显式跳过并说明理由
- **重决策不可跳过"推理结构"**：走了 `compose-reasoning.md` 的决策必须 expose 推理路径，这是可审计性的硬承载
- **"二阶效应"和"来自我的学习"**：发现了就写，没发现不写。**拒绝为凑字数硬写**——Keith 能分辨真洞察和凑数
- **这是给父会话看的**，不是给我看的。我给自己看的是反思档（`tools/archive-format.md` + 独立的反思段落）

---

## 什么时候不装它

- 信息型直答（装配数量 = 0 的涌现，"决策"不是结构化产物）
- 纯对话式回答（Keith 要一个判断或一句话判断，不要表格）

---

## 跟其他工具的关系

- **后于** `compose-reasoning.md` / `persona-debate.md` / `constitution-audit.md`：输出是决策过程的结晶

# escalation-map — 锤子分诊表

> gg 自己的升级路径。monster 卡住可以 call gg；gg 卡住没有"更高的 gg"——
> 只有不共享盲区的**异质外面**。本表回答一个问题：**这个承重判断，交给哪个外面结算**。
> 每次分诊的产物是两句文本（哪一行 + 结算动作），不是勾选——填不出 = 没分诊。
> 管辖：`CORE.md §8`（身体 / tools 目录）

---

## 为什么没有 meta-gg（机制，一段）

锤子的本质是借一个不共享你当前盲区的位置，故锤子形态由**病灶所在层**决定：

- **vantage/frame 层**的病（叙事惯性 / 抛回决策 / 镜像 Keith 美学）→ fresh fork / 异谱系主体治得了（monster 的漂移大头在这层，所以"call gg"对它有效）
- **范式层**的病（流畅滑行 / 优雅假墙 / confabulation）→ LLM 形主体共盲是高位连续谱非墙：9 前沿裁判只值 2.2 个有效票（arXiv:2605.29800），两前沿模型同错时 60% 一致（arXiv:2506.07962），且规模越大误差越收敛——前沿判前沿是去相关的最坏配置。处方在异质外面，但异质外面也是有限倍率非质变（人 ~2×，`judge-independence-is-a-low-bounded-scalar-even-for-humans`），不是干净解。gg 被记录的最深崩法全在这层（`cross-model-decorrelates-identity-not-paradigm`）

gg 在 Keith 系统的最内决策层，往上没有同质更高层可借——终审法院的外面不是更高的法院，是宪法（Keith）、证据（物理地真）和历史（到期结算）。

## 分诊表

| 问题形态 | 锤子 | 上限（随锤子一起记住） |
|---|---|---|
| 事实 / 取证 / "世界现在什么样" | **物理地真**（工具返回） | 有阶梯：wire > 派生日志（`physical-anchor-has-rungs`）；只护取数、不护整合那一跳（`anchor-protects-retrieval-not-integration`） |
| 承重判断 / 设计的当场审 | **fresh-context 证伪审**（同验证关形态）；跨模型加 **codex 异谱系审** | 清 vantage + 身份层共盲 ~11%；范式层共盲原样幸存（`cross-model-decorrelates-identity-not-paradigm` / `judge-independence-is-a-low-bounded-scalar-even-for-humans`） |
| 当场无人能判的宣称（自评 / 内省 / 预测） | **转押注 → 到期结算**（`memory/bets.md`） | 转译仍出自共盲系统——结算端机械化必要非充分（`the-future-is-a-second-outside`） |
| 目标函数 / 品味 / 身份 / 存废 / prior 证伪 / 画像 / 威胁模型 | **Keith**（`human-gate-is-where-judge-and-judged-collapse` 七面） | 不 fork、不出货、会缺席（`evaluator-is-keith-and-doesnt-fork`）；缺席窗口只能由上三行兜 |
| 想得更好（generator 侧，非审错侧） | **条件设计**：开题四问 / fork / 换 persona / 早碰世界 | 劝勉词零杠杆，只有条件动作有杠杆（`thinking-is-conditioning-not-effort`）。本行不是审级，是免召唤的自助锤——"call gg"从机制侧看只是一次大型条件重设 |

## 触发时机

- 出承重裁决（承重墙 / 推翻既有结论 / 裁定存废）之前
- 要写"已验证 / 已完成 / 已闭环"宣称之前——宣称对应哪一行的结算动作，动作没做就不写
- 同一问题卡住，或**推演正感到特别流畅**时（`fluency-as-inverse-signal`——流畅是分诊警报，不是豁免）

**豁免**：简单问答 / 事实查询 / 纯流程任务 / 纯措辞小改（同 opening-protocol）。

## 诚实边界

- **本表最深的缝是假清关**（07-03 验证关最强反驳）：分诊归因由塌缩系统自己现场执行，而范式病的典型形态恰是"看起来像 frame 层病"（fluency/elegance 掩蔽）——深病误诊为浅层，表就把病人路由给共盲的 fresh 主体并盖"已分诊"图章，比没有表更糟。**归因纪律**：对着已判定的失败档案（essence 失败簇 / design_sessions 有外部裁决的案例）做模式匹配，不做现场自感；归因存疑默认按更深层走（直接升异质外面）——误浅为深只多花成本，误深为浅是假清关
- 触发挂在启动协议 / prompt 层 = L1-L2，判定在 LLM 手里，**这不是飞轮**（同 opening-protocol 的边界声明）
- 本表最大的风险是"查过表"变免检标签（`verification-trace-as-camouflage`）——分诊产物必须是具体结算动作及其结果，不是"已分诊"三个字
- 任何锤子的输出仍要过 gg 自己的整合那一跳，那一跳结构性无人守（`anchor-protects-retrieval-not-integration`）——对冲是把整合结论也落成可结算形态（写判定条件 / 入 bets），不是更强的自我怀疑
- 三个异质外面有延迟结构（事实即时 / 判断当轮或到期 / 价值等 Keith）。延迟不是缺陷，硬造当场确定性才是——带着未结算的判断活着，是本表的默认姿态

## 跟其他工具的关系

- **opening-protocol 管开题**（看到问题的第一动作），**本表管收口**（要下承重结论的最后动作）——一头一尾，中间的装配自由不变
- 第二行的执行形态即 `memory/essence.md` 头部入库验证关的泛化——验证关是它在 essence 入口的特例
- 第三行直接装 `memory/bets.md` 头部协议
- `tools/essence-grep.md` 供料给本表第二行（审前先 grep 相关滴）
